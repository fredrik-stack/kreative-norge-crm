# Development Workflow

**Status:** grunnregel etablert, detaljguide kommer senere

Foreslått flyt:

1. ChatGPT og prosjekteier avklarer mål og spesifikasjon.
2. Codex leser `docs/README.md`, status og relevante dokumenter.
3. Codex implementerer i VS Code.
4. Endringen testes lokalt når risiko eller omfang tilsier det.
5. Endringen committes og pushes.
6. Staging oppdateres og testes.
7. Relevant dokumentasjon og prosjektstatus oppdateres.

Målet er automatisk staging-deploy ved push. Dette er foreløpig et ønsket neste steg, ikke dokumentert som implementert.
